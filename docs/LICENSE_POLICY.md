# KERNUL License Policy

Status: Binding  
Scope: Entire repository  
Purpose: Define licensing posture and contribution expectations

## License

KERNUL is released under The Unlicense.

The project is dedicated to the public domain.

Anyone may:

- copy
- modify
- publish
- compile
- distribute
- sell
- use

the software for any purpose.

No attribution is required.

No permission is required.

No warranty is provided.

## Design intent

KERNUL is infrastructure software.

The licensing model is intentionally minimal to:

- remove legal friction
- allow unrestricted experimentation
- permit commercial use
- permit redistribution
- simplify adoption
- avoid licensing lock-in

## Contributor agreement

By submitting code to the repository, the contributor agrees:

1. The contribution may be distributed under The Unlicense.
2. The contributor has the legal right to submit the code.
3. The contributor understands the code will be placed into the public domain.

Contributions that cannot be legally dedicated to the public domain must not be submitted.

## Third-party code

Third-party code must:

- retain its original license
- be clearly identified
- include its license text
- be compatible with redistribution

Do not copy code from:

- proprietary software
- incompatible licensed projects
- unknown sources
- generated content with unclear provenance

## Use of AI-generated code

The use of Agentic tools (AI-generated code) is legally treated the same as human-written code.

All code committed to the repository must be:

- reviewable
- auditable
- traceable
- license-compatible

No code may be committed without explicit human review.

## Governance

The KERNUL project maintainers retain authority to:

- reject contributions
- remove code with unclear licensing
- enforce repository licensing rules
